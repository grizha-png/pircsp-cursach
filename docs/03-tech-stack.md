# Выбор программного стека

## 1. Цели выбора стека

Стек должен был удовлетворять следующим критериям:

- поддержка fullstack CRUD-сценариев;
- простота локального запуска и проверки;
- наглядность для защиты курсовой;
- возможность контейнеризации;
- удобство реализации авторизации и ролевой модели;
- пригодность для небольшого учебного проекта.

## 2. Выбранный стек

### Backend

- **Python 3.12**
- **FastAPI**
- **Uvicorn**
- **sqlite3** из стандартной библиотеки Python
- **Pydantic** для валидации моделей

### Frontend

- **HTML5**
- **CSS3**
- **Vanilla JavaScript (ES Modules не требуются)**
- **Nginx** как контейнер для раздачи статического клиента и проксирования API в Docker-сценарии

### Инфраструктура

- **Docker**
- **docker-compose**

### Тестирование

- **pytest**
- **FastAPI TestClient**

## 3. Почему не выбран более тяжелый стек

### Почему не Spring Boot

- выше порог входа для небольшого учебного проекта;
- больше шаблонного кода;
- сложнее быстро показать всю систему целиком.

### Почему не React/Vue

- для данного проекта ключевая цель — показать архитектуру, CRUD, ролевую модель и API;
- отдельный framework с этапом сборки добавил бы лишнюю сложность;
- интерактивный интерфейс конструктора можно реализовать на чистом JavaScript без потери учебной ценности.

### Почему не PostgreSQL

- для локального и учебного сценария SQLite достаточно;
- преподавателю проще быстро поднять и проверить проект;
- архитектура при этом остается переносимой на PostgreSQL при дальнейшем развитии.

## 4. Результат выбора

Выбранный стек позволяет:

- реализовать backend на FastAPI;
- создать отдельную клиентскую часть;
- обеспечить работу с БД;
- реализовать аутентификацию и авторизацию;
- контейнеризировать приложение;
- подготовить решение к демонстрации и облачному запуску.

## 5. Вывод

Стек **FastAPI + SQLite + HTML/CSS/JavaScript + Docker** является рациональным для данной темы, потому что обеспечивает баланс между скоростью разработки, прозрачностью архитектуры и полнотой выполнения требований курсового задания.
